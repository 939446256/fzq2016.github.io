UIKit-View

UIView（控件）

常用属性

//superview：获得自己的父控件对象,每个子控件的superview只有一个
@property(nonatomic,readonly) UIView *superview;


//subviews：获得自己的所有子控件对象
@property(nonatomic,readonly,copy) NSArray *subviews;


//tag：控件的ID(标识)，父控件可以通过tag来找到对应的子控件;tag值是随意设置，很容易混乱，能不用就不用
@property(nonatomic) NSInteger tag;


//是否可与用户交互
@property(nonatomic,getter=isUserInteractionEnabled) BOOL userInteractionEnabled;


//裁剪超出范围的控件
@property(nonatomic) BOOL  clipsToBounds;

常用方法

// 添加一个子控件view
// 底层实现：
    // 1.判断传入view之前有没有父控件
    // 2.如果有,先移除之前父控件
    // 3.再重新添加view
- (void)addSubview:(UIView *)view;

// 从父控件中移除
- (void)removeFromSuperview;

// 根据一个tag标识找出对应的控件（一般都是子控件）
- (UIView *)viewWithTag:(NSInteger)tag;

viewWithTag:内部的大致实现思路

@implementation UIView
- (UIView *)viewWithTag:(NSInteger)tag
{
    if (self.tag == tag) return self;

    for (UIView *subview in self.subviews) {
        return [subview viewWithTag:tag];
    }
}
@end

UIViewContentMode属性（只对UIImageView控件有效）

  • 带有scale单词的：图片有可能会拉伸

    • UIViewContentModeScaleToFill
      • 将图片拉伸至填充整个imageView
      • 图片显示的尺寸跟imageView的尺寸是一样的
    • 带有aspect单词的：保持图片原来的宽高比
      • UIViewContentModeScaleAspectFit
        • 保证刚好能看到图片的全部(长边伸缩至与父控件一致）
      • UIViewContentModeScaleAspectFill
        • 拉伸至图片的宽度或者高度跟imageView一样（短边伸缩至与父控件一致）

  • 没有scale单词的：图片绝对不会被拉伸，保持图片的原尺寸

    1. UIViewContentModeCenter
    2. UIViewContentModeTop
    3. UIViewContentModeBottom
    4. UIViewContentModeLeft
    5. UIViewContentModeRight
    6. UIViewContentModeTopLeft
    7. UIViewContentModeTopRight
    8. UIViewContentModeBottomLeft
    9. UIViewContentModeBottomRight

UIView的封装(自定义控件）

  1. 自定义控件基础：父子控件

     1. 每个控件都是个容器，能容纳其他控件
     1. 内部小控件是大控件的子控件
     2. 大控件是内部小控件的父控件

  2. 自定义控件的意义

      1. 如果一个view内部的子控件比较多，一般会考虑自定义一个view，把它内部子控件的创建屏蔽起来，不让外界关心

  2. 外界可以传入对应的模型数据给view，view拿到模型数据后给内部的子控件设置对应的数据

  3. 步骤：

    1. 新建一个继承UIView的类

    2. 创建子控件并布局子控件（纯代码或者xib方式）

      1. 纯代码方式

         1. 提供类方法及对象方法
     1. 可以重写init或initWithFrame方法
     2. 重写init创建时，底层也会调用nitWithFrame方法，先调用父类的，再调用子类自身；
 2. initWithFrame:方法中添加并初始化子控件
 3. layoutSubviews方法中布局子控件，布局前必须调用[super layoutSubviews];

      2. xib方式

         1. 新建一个xib文件（xib的文件名最好跟控件类名一样）
     1. 添加子控件、设置子控件属性
     2. 修改最大的父控件的class为控件类名
     3. 将子控件进行连线
 2. 提供一个类方法及对象方法用于加载xib
     1. xib的加载，初始化时底层不调用initWithFrame:方法，而是调用`initWithCoder:方法。
     2. xib加载时，initWithCoder是第一个执行的方法
     3. -(id)awakeAfterUsingCoder:(NSCoder *)aDecoder是第二个执行的方法
     3. awakeFromNib是最后执行的方法，可在这里初始化控件

    3. 提供模型属性，重写模型属性的set方法

      • 在set方法中给子控件设置数据

渐变式动画

  1. 头尾式

    [UIView beginAnimations:nil context:nil];
[UIView setAnimationDuration:2.0];
/* 需要执行动画的代码 */
[UIView commitAnimations];

  2. block式（实质是苹果对头尾式的封装）

      //在1秒内逐渐显示指示版
  [UIView animateWithDuration:1.0 animations:^{
      self.hudLbl.alpha = 1;//不透明
  } completion:^(BOOL finished) {
      //上一个动画完毕后延迟3秒后，在1秒内执行下列动画
      [UIView animateWithDuration:1.0 delay:3.0 options:kNilOptions animations:^{
          //显示指示版
          self.hudLbl.alpha = 0;//完全透明
      } completion:nil];
  }];

UIView界面跳转

  1. 界面间的跳转就是一层一层叠起来的View，这是指如果2个控件如果有重叠部分，那么处于上面的那个控件会盖住下面的。

  2. UIView界面跳转的实质是重新调整窗口subView中元素的顺序，最后一个元素首先显示

  3. 界面跳转主要的方法有3种：

    1. 用 UINavigationController 把 View B push 进来。
      [self.navigationController pushViewController:nextView animated:YES];
    2. 用Modal的 presentModalViewController 方法把 View B 盖在上面。
      [self presentModalViewController:nextView animated:YES];
    3. 把 View A 和 View B 都用 addSubView 加到 AppDelegate 类的 self.window 上。然后调用 bringSubviewToFront 把 View B 显示出来;
      //AppDelegate.m 类
[self.window addSubview:viewB];
[self.window addSubview:viewA];
// 在需要时调用
[self.window bringSubviewToFront:viewB];
